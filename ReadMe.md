Пет-проект: парсер вакансий на разных сервисах
стэк: голанг (пока консольная реализация)

что сделано:

- создан каркас проекта
- реализован механизм поиска вакансий на 2х источниках (HH.ru, SuperJob.ru)
- реализован sharded inmemory cache - для поиска
- реализован sharded inmemory cache - для получения конкретной ваканси по ID из первого кэша
- реализован rate limiter
- сделан адекватный поиск по ID среди результатов

перспектива:

- сделать механизм получения данных при запросе ID, если в основном кэше данных нет
- реализовать паттерн "семафор" для каждого парсера (для ограничения параллельности)
- реализовать circuit breaker (для блокировки недоступного сервиса)
- реализовать глобальный семафор - ограничение одновременных поисков [в parse manager]
- реализовать приоритетные очереди - QoS для разных типов пользователей (платный/админ, обычный, боты) [в parse manager]
- реализовать Load balancer - распределение между источниками [в parse manager]
- реализовать глобальный Circuit Breaker - мониторинг общего здоровья [в parse manager]
- реализовать Resource manager - контроль памяти/CPU [в parse manager]
- реализовать очередь вызовов парсеров - ограничение параллельных вызовов [в parse manager]
- отредакторить код (разделить полностью на слои: DTL, service, repository)
- пагинация результатов поиска
- логирование
- тесты

Планируемая архитектура:
Пользовательский запрос
↓
┌─────────────────────────────────┐
│ PARSER MANAGER УРОВЕНЬ ← Глобальный контроль
│ • Глобальный семафор
│ • Приоритетная очередь
│ • Load balancer между парсерами
│ • Общий circuit breaker
└─────────────────────────────────┘
↓
┌─────────────────────────────────┐
│ PARSER УРОВЕНЬ (HH/SuperJob) ← Индивидуальный контроль
│ • Индивидуальный семафор
│ • Rate limiter источника
│ • Circuit breaker источника
└─────────────────────────────────┘
